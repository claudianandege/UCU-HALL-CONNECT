 Project Overview

UCU Hall Connect is a web-based system designed for Uganda Christian University (UCU) to automate hall room allocation and provide students with a structured platform to submit and track hostel-related complaints (service requests). The system to be built using React (frontend), Express.js (backend), and MySQL (database).

Problem Statement

Hostel room allocation and maintenance reporting are currently handled manually, leading to delays, inefficiencies, poor tracking of complaints, and unfair or unorganized room assignments. Specific issues include:


Manual room assignment is slow
Lack of transparency in allocation
Students struggle to report hostel issues
Complaints are not tracked properly
Poor communication between students and hostel management


Goal

To develop a system that automates hostel room allocation and provides a structured platform for students to submit and track hostel-related complaints.

Why This Project


Improve fairness and transparency in room allocation
Reduce manual workload for hostel administrators
Allow students to report issues easily
Improve response time for maintenance problems
Centralize hostel management processes


Scope

The system is limited to UCU hostels and does not extend to external housing facilities, payment processing, mobile apps, or automated sensor-based monitoring.

Key Features

Student Functions


Register and log in
View room allocation
Submit complaints (e.g., water, electricity, furniture issues)
Track complaint status (Pending → In Progress → Resolved)


Admin/Hostel Manager Functions


Assign rooms to students
Manage room availability
View and update complaints
Mark complaints as resolved


System Functions


Store student and room data
Automatically track room occupancy
Manage complaint lifecycle


Tech Stack


Frontend: React
Backend: Express.js (Node.js)
Database: MySQL
